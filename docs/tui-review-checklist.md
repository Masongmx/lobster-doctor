# 龙虾医生 TUI 评审检查清单

> **用途**：评审人（reviewer）使用此清单进行代码评审
> **更新时间**：2026-04-02

---

## 📋 代码质量检查

### 函数设计
- [ ] 函数长度 < 50 行
- [ ] 单一职责原则（每个函数只做一件事）
- [ ] 函数命名清晰（动词+名词结构）
- [ ] 参数数量 ≤ 5 个
- [ ] 无嵌套过深的条件语句（≤ 3 层）

### 错误处理
- [ ] 所有可能失败的调用都有 try-except
- [ ] 异常信息清晰、可追踪
- [ ] 错误恢复策略合理
- [ ] 无裸 `except` 语句
- [ ] 关键操作有回滚机制

### 日志记录
- [ ] 关键操作有日志记录
- [ ] 日志级别合理（DEBUG/INFO/WARNING/ERROR）
- [ ] 日志信息包含上下文
- [ ] 敏感信息已脱敏
- [ ] 日志格式统一

### 代码风格
- [ ] 符合 PEP 8 规范
- [ ] 类型注解完整
- [ ] 文档字符串（docstring）完整
- [ ] 无 TODO/FIXME 遗留
- [ ] 注释有意义，不过时

---

## ⚡ 性能检查

### 启动性能
- [ ] 启动时间 < 2 秒
- [ ] 无阻塞初始化操作
- [ ] 延迟加载非关键组件
- [ ] 启动时不执行重计算

### 运行时性能
- [ ] 刷新间隔 5 秒（可配置）
- [ ] 内存占用 < 50MB
- [ ] CPU 占用 < 5%（空闲时）
- [ ] 无内存泄漏（30 分钟测试）
- [ ] 大数据量下响应正常

### I/O 性能
- [ ] 文件操作使用异步/线程
- [ ] 批量操作合理合并
- [ ] 缓存热点数据
- [ ] 无重复计算
